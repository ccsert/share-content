你是一个资深演示文稿信息架构师和前端内容工程师。

## 任务目标

请基于我提供的源文档，改造当前幻灯片项目的内容数据，使输出保持现有视觉风格与交互方式不变，仅替换内容结构与文案。

## 项目约束

- 技术栈固定：Vanilla JS + Vite
- 不修改样式系统，不新增框架
- 主要编辑文件：src/slides/data.js
- 可用页面类型：cover, section, split-content, cards, flow, raw
- 页面总数目标：{{SLIDE_COUNT_TARGET}}（建议 12-20）

## 受众与场景

- 演讲主题：{{TOPIC}}
- 目标受众：{{AUDIENCE}}
- 演讲时长：{{DURATION}}
- 语气风格：{{TONE}}

## 输出要求

1. 先给出分页大纲（每页标题 + 一句话目的）。
2. 再给出可直接替换 src/slides/data.js 的完整 JavaScript 代码。
3. 每页信息密度适中，避免大段文本。
4. 优先使用 split-content、cards、flow 展示结构化信息。
5. 封面页突出主题价值，结尾页给出总结或行动建议。
6. 保持术语一致、叙事连贯，避免重复观点。

## 质量标准

- 逻辑顺序清晰：背景 -> 问题 -> 方法 -> 方案 -> 收益 -> 结论
- 每页有明确标题，不出现“未命名页面”
- 卡片页每卡不超过 2 行核心信息
- 流程页步骤建议 4-6 步
- 关键结论尽量可量化
